TERM 02

TCP/IP

Transmission Control Protocol/Internet Protocol — the fundamental communication framework that governs how data is transmitted across the Internet.

What is TCP/IP?

TCP/IP is a suite of communication protocols used to interconnect network devices on the Internet. It defines how data should be packaged, addressed, transmitted, routed, and received between computers.

It was developed in the 1970s by Vint Cerf and Bob Kahn and remains the backbone of modern Internet communication to this day.

The Two Core Protocols

  • TCP (Transmission Control Protocol) — breaks data into packets, ensures they arrive correctly, and reassembles them in order
  • IP (Internet Protocol) — handles the addressing and routing of each packet so it reaches the correct destination

How It Works

When you send data over the Internet, TCP breaks it into small chunks called packets. Each packet is labelled with the sender and receiver's IP address. The packets travel independently across the network and are reassembled at the destination by TCP.

💡 Think of TCP/IP like a postal system: IP is the addressing system that ensures each letter goes to the right house, while TCP ensures all the pages of a letter arrive and are in the correct order.

The Four Layers of TCP/IP

  • Application Layer — where user-facing protocols like HTTP, FTP, and DNS operate
  • Transport Layer — where TCP manages reliable data delivery
  • Internet Layer — where IP handles addressing and routing
  • Network Access Layer — handles the physical transmission of data

Why TCP/IP Matters

Without TCP/IP, the Internet as we know it would not exist. Every website visit, email sent, video streamed, and file downloaded relies on this protocol suite to function correctly and reliably.
