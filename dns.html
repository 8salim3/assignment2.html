TERM 06

Domain Name Server

The Internet's phonebook — a system that translates human-friendly domain names into machine-readable IP addresses.

What is a DNS?

The Domain Name System (DNS) is a hierarchical naming system that converts domain names like www.google.com into IP addresses like 142.250.80.46 that computers use to identify each other on the network.

Without DNS, users would have to memorise long numerical IP addresses to visit every website — DNS makes the web human-friendly.

How DNS Works

  • You type a domain name into your browser
  • Your computer asks a DNS resolver for the IP address
  • The resolver queries a series of DNS servers to find the answer
  • The IP address is returned to your browser
  • Your browser connects to the web server at that IP address

Types of DNS Servers

  • DNS Resolver — the first point of contact, usually provided by your ISP
  • Root Name Server — the top level of the DNS hierarchy
  • TLD Name Server — handles top-level domains like .com, .org, .ke
  • Authoritative Name Server — holds the actual DNS records for a domain

DNS Records

  • A Record — maps a domain to an IPv4 address
  • AAAA Record — maps a domain to an IPv6 address
  • MX Record — directs email to the correct mail server
  • CNAME Record — creates an alias from one domain to another

Why DNS Matters

💡 DNS is often called the "phonebook of the Internet." Just like how a phonebook maps names to phone numbers, DNS maps domain names to IP addresses — making the web accessible and easy to navigate.
