TERM 07

Hypertext Transfer Protocol

The foundation of data communication on the Web — a protocol that defines how messages are formatted and transmitted between browsers and servers.

What is HTTP?

HTTP (Hypertext Transfer Protocol) is the set of rules that governs how data is exchanged between a web browser and a web server. Every time you visit a website, HTTP is working behind the scenes to fetch and deliver the content you see.

It was created by Tim Berners-Lee in 1989 and has gone through several versions, with HTTP/2 and HTTP/3 being the most modern standards today.

HTTP vs HTTPS

  • HTTP — data is transferred in plain text, making it vulnerable to interception
  • HTTPS — the secure version, data is encrypted using SSL/TLS before transfer
  • Modern browsers mark HTTP sites as "Not Secure" to warn users
  • HTTPS is now the standard for all websites, especially those handling sensitive data

How HTTP Works

HTTP follows a request-response model. The browser sends an HTTP request to the server specifying what resource it needs. The server processes the request and sends back an HTTP response containing the resource and a status code.

💡 Common HTTP status codes: 200 = OK (success), 404 = Not Found, 500 = Internal Server Error, 301 = Moved Permanently.

HTTP Methods

  • GET — retrieves data from the server
  • POST — sends data to the server (e.g. form submissions)
  • PUT — updates an existing resource on the server
  • DELETE — removes a resource from the server
  • HEAD — like GET but only retrieves headers, not the body

Why HTTP Matters

HTTP is the language that browsers and servers use to communicate. Without it, the Web simply could not function. Every page load, form submission, and API call relies on HTTP to transfer data reliably and consistently.
